Omnissa Dynamic Environment Manager

简化跨桌面和应用程序的用户配置文件、环境设置和策略的管理。

产品简介

Dynamic Environment Manager 为最终用户提供个性化的动态 Windows 桌面。通过 Dynamic Environment Manager，您可以根据用户的角色、设备和位置提供对 IT 资源的访问来自定义桌面。通过这种方式，您可以创建适应用户特定需求的桌面。

Omnissa Dynamic Environment Manager 标准版和企业版

Omnissa Dynamic Environment Manager 提供标准版和企业版。

标准版 专为寻求基本用户环境管理和应用程序控制的组织而设计。标准版可帮助 Omnissa Horizon® 标准版和 Omnissa Horizon® 高级版客户进行用户配置文件管理。

企业版 专为需要高级管理、详细的应用程序分析以及复杂的自动化和集成功能的大型或复杂环境而设计。企业版是 Omnissa Dynamic Environment Manager 的全功能版本。

部署模式

Dynamic Environment Manager 可在独立模式和集成模式下使用。两种模式都使用配置文件进行 Dynamic Environment Manager 的个性化和应用程序配置管理。

独立模式 (Standalone mode) 整合模式 (Integration mode)
当您想要独立配置 Dynamic Environment Manager 时，请使用独立模式。 使用集成模式将 Dynamic Environment Manager 与 Omnissa Workspace ONE® UEM 集成。
配置文件存储在配置 SMB 共享中，该共享是文件服务器上的中央共享。 配置文件包含在 Dynamic Environment Manager 配置文件中。使用 Workspace ONE UEM，您可以将 Dynamic Environment Manager 配置文件文件定位到特定智能组 (sysin)，以将 Dynamic Environment Manager 配置分发到 Workspace ONE UEM 端点。

功能特性

应用配置管理

借助 Omnissa Dynamic Environment Manager，您可以配置应用程序的初始设置，而不必依赖应用程序的默认设置。您可以定义用户可以个性化的应用程序设置以及用户每次打开应用程序时始终保持不变的设置。通过这种方式，您可以将策略强制设置和用户个性化结合起来。

您还可以使用 Dynamic Environment Manager 在应用程序启动时管理某些用户环境设置。例如，您可以配置驱动器和打印机映射、应用文件和文件夹以及注册表的自定义设置以及运行自定义任务 (sysin)。您还可以为所有用户定义设置和配置，以保证合规性并提供一致的环境。

您可以使用 Dynamic Environment Manager Application Profiler 捕获应用程序的预定义设置。您可以在 Application Profiler 监视的参考系统上运行应用程序并配置所需的设置。

用户环境设置

您可以使用 Omnissa Dynamic Environment Manager 集中管理用户执行日常任务所需的用户设置。当用户登录或启动特定应用程序时应用这些设置。

例如，一家跨国公司的最终用户来自多个国家。该公司可以集中管理不同的显示语言、壁纸、键盘配置和其他区域设置。

通过使用 Dynamic Environment Manager，您可以管理以下设置：

  • 基于 ADMX 的设置
  • 应用程序拦截
  • 应用程序快捷方式和文件类型关联
  • 显示语言
  • 驱动器和打印机映射
  • 环境变量
  • 文件、文件夹和注册表设置
  • 文件夹重定向
  • 隐藏驱动器
  • Horizon 智能策略
  • 登录和注销任务
  • 权限提升
  • 触发任务

应用程序和 Windows 的设置个性化

Omnissa Dynamic Environment Manager 是一种配置文件管理功能，使用户能够漫游其个人桌面和应用程序设置。个性化将用户特定的桌面和应用程序设置与 Windows 操作系统分开。这样，这些设置就可以跨操作系统版本、设备和应用程序实例使用。

个性化独立于 Windows 用户配置文件管理工作，并促进虚拟化技术和应用程序交付机制的管理。Dynamic Environment Manager 个性化与本机安装和虚拟化应用程序无缝集成 (sysin)，跨物理或虚拟、本地或远程 Windows 平台提供一致的用户体验。

Dynamic Environment Manager 个性化允许用户创建自己的个人设置。例如，开发人员可能希望根据自己的习惯配置 Eclipse，并在多个环境中应用该配置。质量工程师可能希望将错误跟踪网站设置为所有浏览器的主页。

应用程序设置的迁移

Omnissa Dynamic Environment Manager 可以将用户的个人应用程序设置从一个操作系统传输到另一个操作系统，例如从 Windows 7 传输到 Windows 10。这是 Dynamic Environment Manager 的默认行为，唯一的要求是应用程序必须将其配置存储在同一注册表中和用户配置文件中的 AppData 位置。

Dynamic Environment Manager 还提供基于 XML 的机制 (sysin)，用于应用程序版本之间的设置迁移。这样，您可以避免在应用程序升级过程中个性化数据丢失的情况，因为升级后个性化数据存储在不同的位置。

Dynamic Environment Manager 下载包包含用于在不同版本的 Microsoft Office 之间迁移的 XML 迁移文件示例。

桌面的动态设置

通过使用 Dynamic Environment Manager，您可以根据使用条件动态调整最终用户桌面的内容和外观。Dynamic Environment Manager 包括可供您根据用户、位置和设备特征组合的即用型条件。

例如，通过使用条件，您可以根据用户当前的物理位置提供对网络打印机的访问，或者根据用户的身份在桌面上创建应用程序快捷方式。

当用户解锁其工作站或重新连接到远程会话时，您可以重新评估条件。您可以从 Dynamic Environment Manager 控制台管理条件并将其应用到 Dynamic Environment Manager 中的所有可配置项目。

新增功能

Omnissa Dynamic Environment Manager 2412 | 10 JAN 2025 | Build 10.14.0.2237

  • Omnissa Dynamic Environment Manager。作为向 Omnissa 过渡的一部分，Dynamic Environment Manager (DEM) 已得到更新。您将注意到所有 DEM 套件组件的视觉更新，包括管理控制台、SyncTool、Application Profiler、Helpdesk 支持工具和产品安装程序。

  • 版本历史。配置版本控制和恢复是一项强大的功能，可提供对配置文件的灵活性和控制 (sysin)。它简化了意外更改的恢复过程，提供了一种可靠的方法来跟踪和管理配置更新。请参阅版本历史记录。

  • 帮助。为了方便搜索帮助，DEM 管理控制台和应用程序分析器现在有一个管理指南链接。单击“管理指南”按钮将转到该产品的管理手册。

  • Horizon 智能政策。为了与 Horizo​​n Agent 更改保持一致，修改了有关 Blast 协议的两个 Horizo​​n 智能策略设置。“最大帧速率”支持增加至 120 fps。策略列表不再包含 Switch Encoder。

  • 运行一次标志文件增强。新的配置设置允许您控制后备.flag文件的创建，从而为您的工作流程提供灵活性和自定义。默认情况下，系统保持现有行为，确保与当前设置的兼容性并防止安装基础中断。但是，通过此新设置，您可以在不需要时快速关闭后备文件的创建，从而减少混乱并符合您的操作要求。

  • 最新操作系统支持。Dynamic Environment Manager 现在支持 Windows 11 24H2 和 Windows Server 2025。

  • 更新了组件。此版本更新了所有组件，包括主要组件、Application Profiler、Helpdesk 支持工具和 SyncTool。
